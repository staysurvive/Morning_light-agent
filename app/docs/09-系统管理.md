# 模块九：系统管理

## 路由

```
/system/users      → 用户管理
/system/roles      → 角色权限
/system/api-keys   → API Key管理
/system/audit      → 审计日志
/system/alerts     → 告警规则
/system/settings   → 系统配置
```

```typescript
interface User {
  id: string
  username: string
  email: string
```

### 告警数据结构

```typescript
interface AlertRule {
  id: string
  name: string
  description: string
  condition: {
    metric: string                // 监控指标
    operator: 'gt' | 'lt' | 'eq' | 'gte' | 'lte'
    threshold: number             // 阈值
    duration: number              // 持续时间(分钟)
    scope?: string                // 作用范围(Agent ID等)
  }
  notification: {
    channels: ('email' | 'webhook' | 'sms')[]
    recipients: string[]          // 通知接收人
  }
  status: 'enabled' | 'disabled'
  createdBy: string
  createdAt: string
  updatedAt: string
}

interface ActiveAlert {
  id: string
  ruleId: string
  ruleName: string
  level: 'info' | 'warning' | 'error' | 'critical'
  message: string
  triggeredAt: string
  duration: string                // 持续时间
  status: 'active' | 'acknowledged' | 'resolved'
  resolvedAt?: string
  resolvedBy?: string
}
```

## 六、系统配置 `/system/settings`

### 配置分组

#### 基本设置
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 系统名称 | string | 辰光Agent平台 | 系统显示名称 |
| 系统描述 | string | - | 系统描述信息 |
| 系统Logo | file | - | 系统Logo图片 |
| 默认语言 | select | 中文 | 系统默认语言 |

#### API配置
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 默认模型 | select | GPT-4 | 系统默认使用的模型 |
| 默认Temperature | number | 0.7 | 默认Temperature |
| 默认Max Tokens | number | 4096 | 默认最大Token数 |
| 请求超时 | number | 30 | API请求超时(秒) |
| 最大重试次数 | number | 3 | 失败重试次数 |

#### 存储配置
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 向量数据库类型 | select | Pinecone | 向量数据库 |
| 向量数据库地址 | string | - | 连接地址 |
| 文件存储类型 | select | 本地 | 文件存储方式 |
| 最大上传大小 | number | 50 | 单文件最大MB |

#### 安全配置
| 配置项 | 类型 | 默认值 | 说明 |

| 启用双因素认证 | boolean | false | 2FA |

| 密码最小长度 | number | 8 | 字符 |
| 登录失败锁定 | number | 5 | 次数 |
| IP白名单 | string[] | [] | 允许的IP |

#### 通知配置
| 配置项 | 类型 | 默认值 | 说明 |

| 发件人邮箱 | string | - | 发件邮箱 |

## Mock数据要点

### 用户列表（至少10个）
- 不同角色分布
- 不同状态
- 合理的中文姓名

### 角色列表（4个）
- 管理员、开发者、运营、只读
- 每个角色有不同的权限矩阵

### API Key列表（至少5个）
- 不同权限范围
- 不同状态

### 审计日志（至少50条）
- 覆盖所有操作类型
- 覆盖所有模块
- 时间分布在最近7天

### 告警规则（至少5条）
- 不同监控指标
- 不同状态

### 活跃告警（2-3条）
- 不同级别
- 不同持续时间
