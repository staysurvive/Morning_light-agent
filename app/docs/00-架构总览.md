# 辰光Agent平台 — 架构总览

## 一、产品定位

辰光Agent平台是一个企业级AI Agent管理平台，提供Agent全生命周期管理、知识库管理、Prompt工程、模型管理、工具编排、对话日志分析等能力。

## 四、数据服务层设计（Mock/API 切换）

### 核心思路

所有页面通过 `src/services/` 下的服务函数获取数据，不直接引用Mock数据。服务层内部根据环境变量决定走Mock还是真实API。

```typescript
// src/services/config.ts
export const USE_MOCK = import.meta.env.VITE_USE_MOCK !== 'false'

// src/services/agent.ts
import { USE_MOCK } from './config'
import * as mockAgent from './mock/agent'
import * as apiAgent from './api/agent'

const service = USE_MOCK ? mockAgent : apiAgent
export const getAgentList = service.getAgentList
export const getAgentDetail = service.getAgentDetail
// ...
```

### 切换方式

```bash
# 使用Mock数据（默认）
VITE_USE_MOCK=true npm run dev

# 连接真实后端
VITE_USE_MOCK=false VITE_API_BASE=http://localhost:8080/api npm run dev
```

## 八、开发约定

### 文件命名
- 页面组件：PascalCase，如 `AgentList.tsx`
- 服务文件：camelCase，如 `agent.ts`
- 类型文件：camelCase，如 `types.ts`

### 组件结构
- 每个页面是一个独立组件
- 复杂页面拆分子组件到同级 `components/` 目录
- UI组件统一使用 Shadcn

### Mock数据
- 所有Mock数据带有合理的中文内容
- 数据量足够展示分页效果（每个列表至少15-20条）
- 状态分布合理（不全是成功状态）
- 时间数据使用相对时间（"2分钟前"、"1小时前"）
