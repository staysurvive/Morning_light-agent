# 模块七：对话日志

## 路由

```
/conversations      → 对话日志列表
/conversations/:id  → 对话详情/链路追踪
```

### 表格字段

| 字段 | 说明 |
|------|------|
| 会话ID | 唯一标识 |
| Agent | 对话的Agent名称 |
| 用户 | 用户标识 |
| 对话轮数 | 来回对话的轮数 |
| 状态 | 进行中 / 完成 / 异常 |
| Token消耗 | 本次对话的Token总量 |
| 耗时 | 总响应时间 |
| 满意度 | 用户评价（👍/👎/未评价） |
| 标注状态 | 未标注 / 已标注 / 需优化 |
| 开始时间 | 对话开始时间 |
| 操作 | 查看详情、标注、导出 |

### 筛选条件
- Agent筛选（下拉多选）
- 状态筛选（进行中/完成/异常）
- 时间范围（今天/昨天/最近7天/自定义）
- 用户ID搜索
- 关键词搜索（搜索对话内容）
- 标注状态（未标注/已标注/需优化）
- 满意度（满意/不满意/未评价）

### 对话数据结构

```typescript
interface Conversation {
  id: string
  agentId: string
  agentName: string
  userId: string
  status: 'active' | 'completed' | 'error'
  turns: ConversationTurn[]
  totalTokens: number
  totalCost: number
  totalDuration: number           // ms
  satisfaction?: 'positive' | 'negative' | null
  annotation?: ConversationAnnotation
  startedAt: string
  endedAt?: string
}

interface ConversationTurn {
  id: string
  role: 'user' | 'assistant'
  content: string
  timestamp: string
  trace?: TurnTrace               // 链路追踪信息
}

interface TurnTrace {
  steps: TraceStep[]
  inputTokens: number
  outputTokens: number
  totalDuration: number           // ms
  model: string
}

interface TraceStep {
  type: 'receive' | 'retrieval' | 'tool_call' | 'prompt_build' | 'llm_call' | 'response'
  name: string
  duration: number                // ms
  detail: Record<string, unknown> // 步骤详情
}

interface ConversationAnnotation {
  rating: number                  // 1-5
  tags: string[]                  // 标签
  notes: string                   // 备注
  annotatedBy: string
  annotatedAt: string
}
```

## Mock数据要点

### 对话列表（至少30条）

需要覆盖：
- 不同Agent的对话
- 不同状态（进行中、完成、异常）
- 不同轮数（1-10轮）
- 不同满意度
- 不同标注状态
- 时间分布在最近24小时内

### 对话详情（至少5条完整对话）

每条对话需要：
- 3-8轮完整的对话内容（中文，内容合理）
- 每轮的链路追踪信息
- 知识库检索结果（如果Agent启用了RAG）
- 工具调用记录（如果Agent启用了工具）
- Token统计和费用
- 部分对话有标注信息
