# 删除确认对话框更新说明

## 更新概述

将所有页面的删除确认从浏览器默认的 `confirm()` 弹窗替换为 Shadcn UI 的 AlertDialog 组件，提供更美观、一致的用户体验。

## 新增组件

### AlertDialog 组件
- **文件路径**: `src/components/ui/alert-dialog.tsx`
- **基于**: @radix-ui/react-alert-dialog
- **功能**: 提供模态对话框，用于确认危险操作

### 组件特性
- 遮罩层背景（黑色半透明）
- 平滑的进入/退出动画
- 响应式设计
- 键盘导航支持（ESC关闭）
- 可自定义标题、描述和按钮

## 实现模式

### 1. 状态管理
每个页面添加两个状态：
```typescript
const [deleteDialogOpen, setDeleteDialogOpen] = useState(false);
const [itemToDelete, setItemToDelete] = useState<string | null>(null);
```

### 2. 删除按钮更新
从直接调用删除函数改为打开对话框：
```typescript
// 旧方式
<Button onClick={() => handleDelete(item.id)}>
  <Trash2 />
</Button>

// 新方式
<Button onClick={() => {
  setItemToDelete(item.id);
  setDeleteDialogOpen(true);
}}>
  <Trash2 />
</Button>
```

### 3. 删除函数更新
从同步确认改为异步状态管理：
```typescript
// 旧方式
const handleDelete = async (id: string) => {
  if (confirm('确定要删除吗？')) {
    await service.delete(id);
    reload();
  }
};

// 新方式
const handleDelete = async () => {
  if (!itemToDelete) return;
  try {
    await service.delete(itemToDelete);
    reload();
  } catch (error) {
    console.error('删除失败:', error);
  } finally {
    setDeleteDialogOpen(false);
    setItemToDelete(null);
  }
};
```

### 4. AlertDialog组件使用
```typescript
<AlertDialog open={deleteDialogOpen} onOpenChange={setDeleteDialogOpen}>
  <AlertDialogContent>
    <AlertDialogHeader>
      <AlertDialogTitle>确认删除</AlertDialogTitle>
      <AlertDialogDescription>
        确定要删除这个项目吗？此操作无法撤销。
      </AlertDialogDescription>
    </AlertDialogHeader>
    <AlertDialogFooter>
      <AlertDialogCancel>取消</AlertDialogCancel>
      <AlertDialogAction 
        onClick={handleDelete} 
        className="bg-destructive text-destructive-foreground hover:bg-destructive/90"
      >
        删除
      </AlertDialogAction>
    </AlertDialogFooter>
  </AlertDialogContent>
</AlertDialog>
```

## UI/UX 改进

### 视觉效果
- **遮罩层**: 黑色半透明背景，聚焦用户注意力
- **动画**: 平滑的淡入淡出和缩放效果
- **按钮样式**: 
  - 取消按钮：outline样式
  - 删除按钮：红色背景（destructive variant）

### 交互体验
- **键盘支持**: ESC键关闭对话框
- **点击遮罩**: 点击外部区域关闭对话框
- **焦点管理**: 自动聚焦到取消按钮（安全默认）
- **状态反馈**: 删除过程中的loading状态

### 文案设计
- **标题**: 统一使用"确认删除"
- **描述**: 
  - 明确说明删除的对象
  - 强调操作的不可逆性
  - 对于重要数据，说明删除的影响范围
- **按钮**: 
  - 取消：中性操作
  - 删除：危险操作，使用红色

## 一致性保证

### 命名规范
- 对话框状态：`deleteDialogOpen`
- 待删除项：`{entity}ToDelete`（如 `agentToDelete`, `userToDelete`）
- 删除函数：`handleDelete`

### 样式规范
- 删除按钮统一使用 `bg-destructive` 背景色
- 对话框宽度：`max-w-lg`
- 间距：使用 `gap-4` 和 `space-y-*`

### 错误处理
- 所有删除操作都包含 try-catch
- 失败时在控制台输出错误
- 无论成功失败都关闭对话框并清理状态
